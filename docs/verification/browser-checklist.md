# 手動クロスブラウザ チェックリスト（Wave 0）

「**何をして（操作）→ どうチェックするか（期待結果）**」を 1 行ずつにした実機確認表。
Chrome / Safari / Firefox の 3 ブラウザで実施し、`✅ / ⚠️ / ❌` を記入する。
プロトタイプはビルド不要、ファイルを直接開く。

```bash
open proto/print-test.html       # A 印刷検証（Task 0.1）
open proto/vertical-test.html    # B 組版検証（Task 0.2）
```

判定の凡例: ✅ 期待どおり / ⚠️ 一部崩れ・要メモ / ❌ 不成立（design 反映 or ROADMAP 送り判断）

---

## A. 印刷検証 — `proto/print-test.html`（Task 0.1）

| # | 何をして（操作） | どうチェックするか（期待結果） | Chrome | Safari | Firefox |
|---|------------------|--------------------------------|:------:|:------:|:-------:|
| A1 | ページを開く | 右側に縦書き本文、左に textarea、上にヘッダの 2 ペインが出る | | | |
| A2 | 「A5」ボタンを押す | `<body data-paper>` が `a5` になり、画面のサイズ目安が A5 になる | | | |
| A3 | A5 のまま印刷プレビューを開く（⌘P） | **本文（縦書き）だけ**が出て、ヘッダ・textarea・チェックリストが消える | | | |
| A4 | 印刷プレビューの用紙を確認 | 用紙が **A5**（148×210mm 相当）になっている | | | |
| A5 | 「B6」ボタンを押して再度印刷プレビュー | 用紙が **B6**（128×182mm 相当）に切り替わる | | | |
| A6 | 本文が長い状態でプレビューのページ送りを見る | 2 ページ目以降へ縦書き本文が**右→左に正しく流れる**（切れ・重なりなし） | | | |
| A7 | （Safari 重点）A4/A5 で用紙が変わらない場合 | 実寸固定フォールバック（`.tategaki` を A5/B6 実寸）で**本らしい余白・判型**になる | – | | – |
| A8 | PDF に保存して開く | 画面プレビューと印刷で**縦書きの向き・行の流れが一致**する | | | |

> A4/A5 が Safari で ❌（用紙が変わらない）の場合は A7 が成立すれば実用可。両方 ❌ なら US-006 に関わるので design「印刷」節へ要記録。

## B. 組版検証 — `proto/vertical-test.html`（Task 0.2）

| # | 何をして（操作） | どうチェックするか（期待結果） | Chrome | Safari | Firefox |
|---|------------------|--------------------------------|:------:|:------:|:-------:|
| B1 | ページを開く | 本文が縦書き（右→左）で表示される | | | |
| B2 | 句読点・括弧の位置を見る | 行頭に `。、」`、行末に `「（` が**来ない**（`line-break: strict` の禁則が効く） | | | |
| B3 | 半角の 2 桁数字・西暦・英単語を見る | `12` `2026` `OK` `CSS` 等の横倒しが**読める許容範囲**か（縦中横なし前提） | | | |
| B4 | 段落の頭を見る | 各段落の冒頭が **1 字下げ**（`text-indent`）されている | | | |
| B5 | 段落どうしの間隔を見る | 段落間に**適切なアキ**があり、本文中の単一改行（`<br>`）と区別できる | | | |
| B6 | 行の両端（上下）を見る | `text-align: justify` で**行末が揃う**（均等割り付け） | | | |
| B7 | 段落最終行・短い行を見る | 短行が**不自然に引き伸ばされて崩れていない**（崩れたら要件緩和済みなので記録のみ） | | | |

> B3（英数字）が ❌ → 縦中横（ROADMAP）送りを検討。
> B4/B5（字下げ・アキ）→ `text-indent` のみか `margin-inline-start` 併用かを判断し design「CSS 組版方針」に反映。
> B6/B7（justify）→ US-003 で既にベストエフォート化済み。結果を design に記録するだけでよい。
