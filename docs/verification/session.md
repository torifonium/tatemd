## ▶️ ステージ3: 組版プロトタイプ（次はここ）

**何を**: `proto/vertical-test.html` を **まず Chrome（第一対象）** で開き、縦書きの和文組版が「本として読める」かを確認する。Safari / Firefox は可能なら見て、崩れは issue 化（最初から完全カバーしない）。

**なぜ重要か**: 禁則・約物・英数字・段落・均等割りを **CSS にネイティブ委譲**し、core(JS) では一切補正しない設計。だからこそ「ブラウザ任せで本当に本らしく組めるか」をここで見極める。崩れる項目は要件緩和か ROADMAP（縦中横など）送りを判断する分岐点。

具体的に見る点:
- **禁則**（`line-break: strict`）: 行頭に `。、」`、行末に `「（` が来ないか。3 ブラウザで挙動差が出やすい。
- **半角英数字**（縦中横なし）: `12` `2026` `OK` `CSS` が横倒しになる。**読める許容範囲か**。不可なら縦中横（ROADMAP）送り判断。
- **段落**（`breaks:true`）: 単一改行（`<br>`）と空行段落（`<p>`）が共存。字下げ・段落アキが本らしいか。`text-indent` だけで足りるか `margin-inline-start` 併用かを決める。
- **均等割り付け**（`text-align: justify`）: 行末が揃うか、短行で崩れないか（既にベストエフォート化済み・結果記録のみ）。

**成功の状態**: 句読点・括弧が行頭行末で暴れず、段落が見分けられ、英数字が許容範囲で、全体が「本の組版」に見える。

```bash
open proto/vertical-test.html    # まず Chrome（第一対象）
```

**チェック**（詳細は browser-checklist B1〜B7）:
- [ ] B1 本文が縦書き（右→左）で表示される
- [ ] B2 行頭に `。、」`・行末に `「（` が来ない（禁則が効く）
- [ ] B3 半角 2 桁数字・西暦・英単語の横倒しが**読める許容範囲**
- [ ] B4 各段落の冒頭が 1 字下げされている
- [ ] B5 段落間に適切なアキがあり、本文中の `<br>` と区別できる
- [ ] B6 `justify` で行末が揃う
- [ ] B7 段落最終行・短行が不自然に崩れていない

→ 結論（**段落アキ方式** text-indent / margin-inline-start 併用、英数字の許容可否、justify の実効）を `design.md`「CSS 組版方針」節へ反映。
